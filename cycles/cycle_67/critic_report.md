# Cycle 67 — Critic Report (Lenny 인사이트 적용 평가)

## 잘한 점
- **결과 프레이밍**: "14/137 완료" 대신 "결혼 준비 10% 완료!" — 사용자에게 의미 있는 숫자
- **단일 추천**: 결정 피로 제거. "오늘 이것만 하세요" 패턴은 Duolingo의 핵심 전략
- **맥락 제공**: 왜 이 태스크인지(quest 정보), 설명 포함 → "이해 > 마찰" 원칙 적용
- **페이스 피드백**: D-day 기반 속도 판단은 결혼 앱만의 고유 가치

## Trade-off
- SmartHomeSummary + SmartRecommendation 두 컴포넌트가 "추천" 기능에서 겹칠 수 있음
  → 하지만 SmartHomeSummary는 "전체 맥락"에 집중, SmartRecommendation은 "구체적 태스크 목록"에 집중. 역할 분리 OK.
- 페이스 계산이 "12개월 기준"으로 고정 → 6개월 단기 준비 커플에겐 맞지 않을 수 있음
  → 추후 준비 기간 설정 기능으로 보완 가능

- 이 판단은 올바르다고 봄 — 모든 프레임워크를 적용하면 오히려 제품이 흐려짐
