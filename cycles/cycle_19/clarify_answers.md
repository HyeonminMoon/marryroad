# Cycle 19 — Clarify Answers

1. **"애매하다"의 핵심**: 태스크를 클릭했을 때 어디가 디테일이고 어디가 체크리스트인지 경계가 불분명. 인라인 폼이 카드 안에서 펼쳐지면서 레이아웃 시프트 발생, 시각적 계층 부재.

2. **3중 중첩 문제**: 모달(DialogContent) → 태스크 카드(border rounded-lg) → 인라인 폼(border-t pt-4). 사용자 입장에서 "이 폼이 어디에 속하는 건지" 직관적이지 않음. 특히 스크롤이 길어지면 맥락을 잃음.

3. **완료 요약의 문제**: 비용/메모/날짜/업체/평점이 모두 같은 크기 아이콘+텍스트로 나열됨. 시각적 강조 없이 평면적. 메모가 길면 카드가 비대해짐.

4. **정보 우선순위**:
   - 모든 태스크 공통: 완료 날짜, 메모
   - 비용 관련 태스크만: 비용, 업체 정보
   - 선택적: 평점
   → 현재는 `showCost` prop 하나로만 구분. 더 세밀한 섹션 분리 필요.

5. **사이클 연속성**: Cycle 18 learning_report의 1순위 추천이 정확히 "Task detail UI 개선". 사용자 피드백과 에이전트 분석이 일치.
