# Cycle 1 - Clarify Questions

## Problem Space

1. **What's the #1 pain point in the current system?**
   - Is there a feature gap that causes users to leave the app?

2. **What emotional moment is missing?**
   - The current system tracks tasks but doesn't tell a story.
   - Where do emotional milestones fit?

3. **What data does the system collect but NOT use?**
   - completedDate, vendorInfo, rating, photos, memos - all collected per task
   - But there's no visualization or storytelling using this data

4. **Who is the user?**
   - A couple (or one partner) preparing their wedding
   - They need both logistics (what to do next) AND emotional support (celebrating progress)

5. **What's the biggest time-waster in wedding prep?**
   - Coordination between partners (who does what?)
   - Tracking vendor decisions across multiple tasks
   - Not knowing what's coming next after completing something

6. **What would make this app feel "alive" vs just a checklist?**
   - Timeline visualization of the journey
   - Celebration moments when milestones are reached
   - A sense of narrative progression

7. **What's the competitive landscape?**
   - The Knot, WeddingWire: heavy, generic, ad-heavy
   - Notion templates: flexible but no gamification
   - This app: gamified + Korean wedding context = unique niche

8. **What's technically feasible in one cycle?**
   - We have: Zustand store, localStorage, React Flow, Framer Motion
   - We can build a new page/component without backend changes
