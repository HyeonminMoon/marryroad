# Cycle 64 — Critic Report
## 잘한 점
- 타입 시스템을 별도 파일로 분리하여 재사용성 확보
- getStats()에서 +1 게스트까지 자동 집계 — 실제 참석 인원 파악에 유용
- 검색 + 필터 조합으로 대규모 하객 목록도 관리 가능

## Trade-off
- localStorage 기반이라 데이터 용량 한계 (수백 명 수준은 문제 없음, 수천 명은 미지수)
- 축의금 필드가 선택적 → 입력 안 하면 통계가 정확하지 않을 수 있음
  → 하지만 강제 입력은 UX를 해치므로 현재 선택이 맞음

## 개선 여지
- Excel/CSV 내보내기 (L1 이슈와 연결)
- 하객 그룹/테이블 배정 기능
- 축의금 입력 리마인더
