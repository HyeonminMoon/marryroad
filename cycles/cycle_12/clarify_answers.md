# Cycle 12 — Clarify Answers

1. **Korean 2030s UX expectations**:
   - Clean, minimal UI (Notion-like) with smooth animations
   - Mobile-first (most access via phone)
   - Satisfying micro-interactions on actions (check, complete, toggle)
   - Card-based layouts with generous whitespace
   - Soft gradients and rounded corners
   - Quick feedback: haptic-like animations on state changes

2. **Current friction points**:
   - /roadmap page is very dense — too many sections stacked vertically
   - No animation on task completion (just instant state change)
   - Progress bar in sticky header is functional but not emotionally satisfying
   - View toggle (map/path) has no transition animation
   - Cards feel static — no hover/press feedback

3. **Missing micro-interactions**:
   - Task completion animation (confetti for first few, then subtle check animation)
   - Number counting animation on XP/stats
   - Smooth expand/collapse transitions (current ones are instant)
   - Press/hover feedback on interactive cards
   - Level-up celebration (confetti exists but no visual announcement)

4. **Information hierarchy issues**:
   - Heatmap, achievements, budget chart, couple message, D-Day, today section, map — that's 7 sections
   - Users see all of this at once and don't know where to focus
   - Need a "hero section" that grabs attention first

5. **"오 이거 좋다" moments**:
   - Animated progress ring (like Apple Watch activity rings)
   - Smooth number animations
   - Satisfying completion sound/animation
   - A clear "today's focus" that reduces decision paralysis
